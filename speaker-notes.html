1 Title — Stock Portfolio Advisor
~30 seconds
  • Welcome everyone. We're Team Nova Core and this is Stock Portfolio Advisor.
  • The one-line pitch: an agentic AI that thinks about your portfolio so you don't have to.
  • Built in a single hackathon session using Kiro, FastAPI, React, and OpenRouter LLMs.
  • Let me start with the problem we set out to solve.
2 The Problem — Retail investors are flying blind
~60 seconds
  • There are three tools retail investors typically reach for — dashboards, robo-advisors, and human advisors.
  • Dashboards show you numbers but don't explain what they mean or what to do.
  • Robo-advisors give a recommendation but it's a black box — no reasoning shown.
  • Human advisors are expensive, slow, and not available at 11pm when you're anxious about your portfolio.
  • The gap: nobody gives you the score AND the reasoning AND lets you ask follow-up questions. That's what we built.
3 The Solution — Quant scoring + AI reasoning + conversation
~90 seconds
  • Our solution has four layers working together.
  • First, a quant engine scores each stock using beta, P/E ratio, PEG ratio, momentum, and any custom criteria the user sets.
  • Second, an LLM agent fetches live market data — price, news sentiment, SEC filings — reasons about it, and critiques its own output.
  • Third, results are pushed to the dashboard in real time via WebSocket — you see the score and rationale appear live.
  • Fourth, a chat interface lets you ask "why is AAPL flagged?" and get a portfolio-aware answer.
  • The mock on the right shows what a typical output looks like — a risk score of 72 with a plain-English rationale.
4 Technical Deep Dive — The Agentic Loop
~2 minutes
  • This is the most technically interesting part. The agent loop has five stages.
  • Memory: the previous score and rationale are injected into the system prompt so the LLM reasons about what changed, not just the current snapshot.
  • Concurrent tool fetch: yfinance, Finnhub news, and SEC EDGAR all run in parallel via asyncio.gather — no sequential waiting.
  • Structured output: the LLM must emit exactly three fields. A regex parser with three fallback strategies handles noisy or malformed output.
  • Self-reflection: the LLM critiques its own answer. If the score delta exceeds a threshold it revises. It stops early when it converges.
  • Guardrails: 12 regex patterns catch leaked prompt templates, think-blocks are stripped, and there's a score-derived fallback if the rationale is garbage.
  • On the right — an 8-model fallback chain on free-tier OpenRouter. It rotates on 429s with a 1-second backoff.
5 AI-Assisted Development — Built with Kiro
~90 seconds
  • We used Kiro as our AI development partner throughout — not just for autocomplete but for the entire development loop.
  • We started with a one-line idea. Kiro produced a requirements doc, a design doc, and a task list. Then it executed each task with full context.
  • It generated FastAPI routers, React components, Alembic migrations, and property-based tests — all from the spec, not ad-hoc prompts.
  • It caught real bugs in the same turn: a KeyError because Civic Auth uses "id" not "sub", a duplicate function from a mid-edit cutoff, a YAML path resolution issue.
  • 86 unit tests, zero written by hand — all generated from acceptance criteria using Hypothesis strategies.
  • Net result: a two-week sprint scope delivered in a single hackathon session.
6 What We Shipped — Full stack, end to end
~60 seconds
  • On the backend: FastAPI with PostgreSQL, async SQLAlchemy, five Alembic migrations, APScheduler for 30-minute refreshes, Civic Auth JWT, SlowAPI rate limiting, and a WebSocket manager.
  • On the frontend: React 18 with TypeScript, TanStack Query, Recharts for 1w/1y/2y price charts, a slide-out chat drawer, and live WebSocket updates throughout.
  • On quality: 86 passing tests, property-based testing with Hypothesis, output guardrails, think-block stripping, and score-derived fallback rationale.
  • The four stats at the bottom tell the story — 86 tests, 8 LLM fallbacks, 5 DB migrations, 1 session to build it all.
7 Honest Assessment — How good is the agent?
~60 seconds
  • We wanted to be honest about where the agent sits on the capability spectrum.
  • Autonomy is moderate — it runs the loop independently but still needs human-defined criteria and thresholds.
  • Usefulness is high — the combination of score, rationale, and chat is genuinely more useful than any single tool.
  • Technical depth is good — multi-step reasoning, tool use, memory, and self-critique are all real.
  • Creativity is high — the portfolio-level cross-ticker analysis was an emergent capability we didn't explicitly design for.
  • What would push it further: agent-initiated refresh on news spikes, persistent chat history, sector-aware tool selection.
8 Challenges & What's Next
~60 seconds
  • Deployment was our biggest challenge — cross-origin auth, cookie vs Bearer token, and hosting config took longer than building the features themselves.
  • Kiro was excellent at code generation but struggled with live environment debugging and hosting configuration — that's a real limitation of current AI tooling.
  • Free-tier LLMs are fragile — they ignore tool schemas, hallucinate output formats, and wrap responses in think-blocks. Our three-layer fallback parser was essential.
  • API rate limits on yfinance, Finnhub, and OpenRouter all required semaphore and backoff handling.
  • For the future: mobile PWA with push alerts, benchmark comparison vs S&P 500, email/SMS threshold notifications, multi-model routing by task type, and historical backtesting.
